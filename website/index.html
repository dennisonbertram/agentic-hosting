Security

gVisor isolation.
Kernel-level sandbox.

Every container runs inside a gVisor sandbox. Syscall interception at the kernel level means compromised code can never reach the host. Read-only rootfs with dropped capabilities by default.

  • Syscall interception — host kernel never exposed
  • Read-only rootfs by default
  • All Linux capabilities dropped
  • Secure for running untrusted AI-generated code

Databases

Managed databases.
Provisioned in seconds.

Provision Postgres or Redis with a single API call. Encrypted connection strings are returned immediately and automatically injected into your service environment. No connection string wrangling.

  • Postgres and Redis on demand
  • AES-256-GCM encrypted connection strings
  • Auto-injects DATABASE_URL into service env
  • Service automatically restarted after provision

Reliability

Self-healing.
Always in sync.

A background reconciler re-syncs every 60 seconds, restarting drifted containers and cleaning up stale state. A circuit breaker stops crash loops automatically before they consume resources.

  • Reconciler re-syncs every 60 seconds
  • Circuit breaker stops crash loops
  • GC removes stale containers and images
  • Health endpoint returns full system state

Design

Designed for agents,
not humans.

Every API decision is optimized for AI consumption. Idempotency everywhere, structured JSON, no ambiguous states.

Idempotency Keys

Every write operation accepts an idempotency key. Safe to retry — no duplicate deployments.

Per-Tenant Rate Limits

Separate rate limit buckets per tenant. One runaway agent can't starve others.

Structured JSON Everywhere

No HTML, no wizard UIs, no ambiguous states. Every response is machine-readable JSON.

Single Go Binary

One binary, SQLite for state. No Kubernetes, no etcd, no operator. Runs on any Linux box.
